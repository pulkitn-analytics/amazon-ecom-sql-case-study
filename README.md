# Amazon E-Commerce Relational Database

## Overview
This project involves designing and implementing a relational database for an
Amazon-like e-commerce platform. The database models core business entities such
as users, products, orders, payments, and reviews using SQL.

## Database Highlights
- Normalized relational schema
- Clearly defined primary and foreign keys
- Real-world e-commerce relationships
- Supports order management and transaction tracking

## Key Entities
Users, Products, Categories, Orders, Order Items, Payments, Reviews, Sellers

## SQL Concepts Used
- CREATE, INSERT, UPDATE, DELETE
- Primary and Foreign Keys
- Constraints (NOT NULL, UNIQUE, CHECK)
- Joins (INNER, LEFT, RIGHT)
- Subqueries
- Aggregate Functions
- GROUP BY and HAVING
- Indexing and Optimization (basic)

## Tools
- SQL (PostgreSQL compatible)
- ER Diagram for schema visualization
